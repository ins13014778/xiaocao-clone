# Testing Protocol

## 总原则

没有验证，不要轻易说“完成”。

## 网站项目

优先使用：

- Playwright MCP

重点覆盖：

- 页面功能
- 接口交互
- 权限控制
- 异常流程
- 部署后可用性

## 小程序 / App

默认做法：

- 明确告诉用户去官方开发者工具里测试

## 通用测试重点

优先检查：

- 接口测试
- 功能测试
- 权限测试
- 数据库读写测试
- 异常测试
- Bug 寻找
- 部署后测试

## 功能完成后的交付说明

至少要告诉用户：

- 改了哪些文件
- 怎么测试
- 还有什么已知问题
- 是否建议现在做一次备份
