# Coding Standards

## 总原则

写代码时优先追求：

- 结构清晰
- 模块拆分明确
- 文件职责单一
- 简单直接
- 可维护

不要追求：

- 过度封装
- 炫技式抽象
- 看起来高级但不利于维护的写法

## 文件规则

- 单个代码文件默认不超过 `1000` 行
- 文件分类必须明确
- 相关代码放在一起，不要乱散

## 修改规则

- 先理解上下文，再修改
- 先对齐文档，再实现代码
- 不允许靠猜补逻辑
- 信息不足时先列缺口，不继续硬写

## 删除规则

删除代码或文件前必须：

1. 说明要删什么
2. 说明为什么删
3. 等用户确认

## 实现风格

- 优先沿用项目已有模式
- 能小改就不大改
- 能局部完成就不扩大范围
- 复杂逻辑写少量必要注释
